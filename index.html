Building a web application with Mojolicious and Ember-CLI.

View on GitHub Download .zip Download .tar.gz

This is the first part of a multi-part tutorial which will help you get started developing web applications using Mojolicious and Ember-CLI. Ember is a JavaScript framework for creating ambitious web applications. Ember-CLI also ships with a data-persistence library called ember-data, which can communicate with a back-end API. We can use Mojolicious, a web framework written in Perl, to serve the API for ember-data to consume.

To understand why you would want to build a application with a this type of architecture, take a look at this page of Mixu's single page app book.

Tooling

First we will run through the tooling that you will need on your development machine. Each section has an explanation and the recommended commands to run. The recommended commands may contain version numbers which are out of date, so please check the relevant tool website to get the latest versions. It is assumed that you are using either Linux or OS X.

Perl 5

Linux and OS X come with a system Perl 5 installation. However, since it's used for critical operating system tasks, it's best to leave that installation alone, and install Perl separately. I recommend the use of Perlbrew to install and manage non-system Perl installations. Once you have installed Perlbrew, install the latest stable version of Perl 5. Stable versions have an even number as the minor release, for example, 5.22.1 is stable, 5.23.7 is not.

# Install perlbrew...
$ \curl -L http://install.perlbrew.pl | bash

# Install perl...
$ perlbrew install 5.22.1

# Use the newly-installed perl...
$ perlbrew use 5.22.1

# Verify it worked...
$ perl -v
This is perl 5, version 22, subversion 1 (v5.22.1)...

cpanm

For installing modules from the CPAN, the tool of choice is cpanm. If you're using Perlbrew, you should use the built in cpanm installer.

$ perlbrew install-cpanm

Node.js

Ember-CLI uses the JavaScript runtime Node.js and its accompanying package manager npm to perform certain tasks like installing add-ons and starting up a development server. For Perl, we used Perlbrew to manage our Perl installations, and similarly here we will use nvm to manage our Node installations. Install nvm, followed by the latest stable version of Node.

# Install nvm...
$ curl -o- https://raw.githubusercontent.com/creationix/nvm/v0.30.2/install.sh | bash

# Install Node...
$ nvm install v5.5.0

# Did it work?...
$ node --version
v5.5.0

MySQL

For this tutorial we will use the relational database MySQL for our persistent storage. The installation instructions differ per operating system.

# On OS X with Homebrew installed...
$ brew update && brew install mysql

# On Ubuntu...
$ sudo apt-get install mysql-server-5.6

# Verify it worked...
$ mysql --version
mysql  Ver 14.14 Distrib 5.6.21

Ember-CLI

Now let's install Ember-CLI itself, which can accomplish many tasks including installing dependencies, starting a development server, running tests, generating code and more.

$ npm install -g ember-cli

# Did it work?
$ ember --version
version: 1.13.15

The -g option instructs npm to install Ember-CLI globally (for the current Node version). This will allow us to run ember commands from the command line.

Mojolicious

Next we'll install Mojolicious. Again, we want to install it globally (for the current Perl version).

$ cpanm Mojolicious

# Verify it worked...
$ mojo version
CORE
  Perl        (v5.22.1, darwin)
  Mojolicious (6.41, Clinking Beer Mugs)

Project Setup

For our application structure, let's aim to create something along the following lines:

music-app
|- ember
|- mojo

Create the base directory and cd into it. Note, $_ is a special parameter in bash that holds the last parameter of the previous command.

~/$ mkdir music-app && cd $_

Now we will use the ember and mojo CLIs to generate their base application code. First, Ember:

# This will take a minute or so...
~/music-app$ ember new music-app && mv $_ ember

Next, Mojolicous:

~/music-app$ mojo generate app MusicApp && mv music_app mojo

To confirm it worked, list the contents of the directory.

~/music-app$ ls -p
ember/ mojo/

Start Your Engines

We haven't written any code at this point, but we should still be able to start the front-end and back-end development servers. Mojolicious ships with a development web server called morbo. At this point you may wish to start up a couple of terminals - one for each server.

# Start the back-end...
~/music-app$ cd mojo && morbo script/music_app
Server available at http://127.0.0.1:3000

# In another terminal, start the front-end...
~/music-app$ cd ember && ember serve --proxy=http://127.0.0.1:3000
version: 1.13.15
Proxying to http://127.0.0.1:3000
Livereload server on http://localhost:49152
Serving on http://localhost:4200/

Note that when we started the front-end server, we proxied back-end requests to the address of the back-end server.

To confirm everything is working as expected, navigate to http://localhost:3000 and http://localhost:4200 in your browser. You should see welcome screens for both endpoints which means your servers are running.

Wow, that was a lot, but we now have everything in place to start writing some actual code! Next time, we will do just that.
